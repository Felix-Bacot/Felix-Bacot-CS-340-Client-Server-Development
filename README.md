# Felix-Bacot-CS-340-Client-Server-Development

## CS-340 Client/Server Development Portfolio

### Project Overview
This repository contains my completed Project Two dashboard application developed for CS-340 Client/Server Development at Southern New Hampshire University. The project demonstrates the integration of MongoDB, Python, Dash, Plotly, and Dash Leaflet to create an interactive dashboard for Grazioso Salvare. The application enables users to search, filter, visualize, and analyze animal shelter data to identify dogs suitable for search-and-rescue training.

## Repository Contents
- **AnimalShelter.py** – CRUD Python module used to communicate with the MongoDB database.
- **ModuleSixMilestone.ipynb** – Interactive dashboard application.
- **CS 340 7-2 Project Two Submission.docx** – Project documentation and README submitted for Project Two.

# Reflection

## How do you write programs that are maintainable, readable, and adaptable?

I write programs that are maintainable, readable, and adaptable by organizing my code into logical modules, using descriptive variable and function names, and following consistent formatting throughout the project. During Project One, I developed a reusable CRUD Python module that handled all database operations separately from the dashboard application. This modular approach made the code easier to understand, test, and maintain because any changes to the database functionality could be made in a single location without affecting the user interface. Reusing the CRUD module in Project Two also reduced duplicate code and simplified the connection between the dashboard widgets and the MongoDB database. In the future, this CRUD module could be adapted for other database-driven applications, such as inventory management systems, customer relationship management systems, or healthcare databases where reliable Create, Read, Update, and Delete operations are required.

## How do you approach a problem as a computer scientist?

As a computer scientist, I begin by carefully analyzing the client's requirements and breaking the problem into smaller, manageable tasks. For the Grazioso Salvare project, I first focused on designing and testing the MongoDB database connection through the CRUD module before building the interactive dashboard. Once the database functionality was working correctly, I integrated the dashboard components, including the data table, interactive filters, bar chart, and geolocation map. This approach differed from many previous programming assignments because it required integrating multiple technologies into one complete application. In future projects, I will continue using modular design, incremental development, frequent testing, and thorough documentation to develop database solutions that are scalable, reliable, and tailored to client requirements.

## What do computer scientists do, and why does it matter?

Computer scientists develop software solutions that help organizations solve real-world problems by organizing data, automating processes, and supporting informed decision-making. This project demonstrates how software can transform large amounts of raw data into meaningful visual information that users can easily interpret. For Grazioso Salvare, the dashboard allows employees to efficiently search, filter, and visualize animal shelter records to identify dogs that meet specific search-and-rescue training requirements. Automating this process improves efficiency, reduces the time required to analyze data manually, minimizes errors, and enables the organization to make faster, more informed decisions.
